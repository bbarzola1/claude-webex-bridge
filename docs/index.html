How It Works

The bot polls Webex for messages, forwards them to the Claude CLI, and returns responses. No webhooks or public URLs required.

Webex Teams ← poll → Bot (bot.py) ← subprocess → Claude CLI

bot.py

Polling loop, command dispatch, message relay

webex_api.py

Async httpx wrapper for Webex REST API

claude_cli.py

Async wrapper around the Claude CLI subprocess

sessions.py

Session discovery from Claude Code history files

auth.py

Email-based authorization check

config.py

Environment variables and constants

FAQ

Do I need a public URL or ngrok?

No. The bot uses polling (every 2.5 seconds), so it works behind firewalls and NATs without any port forwarding.

Is there any latency?

The polling adds ~2.5 seconds, but Claude CLI calls themselves take seconds to minutes, so it's negligible in practice.

Can multiple people use one bot?

Currently the bot only responds to the email in your .env file. Messages from other emails are silently ignored. This is a security feature.

What if the Claude CLI isn't installed?

The setup script will warn you and provide the install command: npm install -g @anthropic-ai/claude-code. You'll need to run claude once to authenticate before using the bot.

What's the difference between safe and skip-permissions mode?

skip-permissions (default): Claude auto-approves all tool use. Fast but less controlled. safe: Claude asks before using tools, but may hang since approval prompts can't be shown in print mode. Toggle with /safe.

Can I run this on Windows?

Yes. The run.py installer works cross-platform. Run python run.py (or python3 run.py) from the project directory.
